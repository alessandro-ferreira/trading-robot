## Folder Structure

```
.
├── Makefile                # Automates common tasks like running and testing.
├── main.py                 # Application entry point, starts the gRPC server.
├── exchange/               # Implements the gRPC service logic.
│   └── service.py
├── internal/               # Internal helper packages.
│   ├── config.py           # Configuration loading and parsing.
│   └── logger.py           # Structured logging setup.
├── requirements.in         # List of direct Python dependencies.
```

## Getting Started

This guide provides the necessary steps to set up and run the `python-gateway` service.

### 1. Prerequisites

-   Python 3.8+
-   Protobuf Compiler (protoc)

### 2. Installation

It is highly recommended to use a Python virtual environment.

```bash
# From the project root directory (e.g., ~/Documents/Codes/trading)
python3 -m venv <path_to_your_venvs>/venv_trading
source <path_to_your_venvs>/venv_trading/bin/activate
```

Install the required Python packages using the provided `requirements.in` file and `pip-tools`.

```bash
# From the python-gateway/ directory
pip install pip-tools
pip-compile requirements.in
pip install -r requirements.txt
```

### 3. Generate gRPC Code

Use the provided `Makefile` to generate the gRPC server code:

```bash
# From the python-gateway/ directory
make proto
```

### 4. Run the Service

Use the provided `Makefile` to start the gRPC server.

```bash
# From the python-gateway/ directory
make run
```

The server will start and listen for incoming gRPC requests from the `go-bot`.

### 5. Testing

Use the provided `Makefile` to run the tests.

```bash
# From the python-gateway/ directory
make test
```
